# Product

## Register

product

## Users

Anvics is for human developers, maintainers, and technical operators who want a better source-control system for supervising AI coding agents. They are not trying to become merge clerks. They want to see what agents are doing, preserve progress, compare candidate work, recover from failures, and publish trusted results.

The UI audience is human. Agents interact with Anvics through skills, packets, CLI commands, APIs, and workspace projections. Humans use the UI to make higher-level operational decisions: what work is active, what evidence exists, what risk is present, what should be accepted, what needs recovery, and what should be exported or synced.

## Product Purpose

Anvics is agent-native source control. It gives parallel AI coding agents isolated workspaces, durable snapshots, compact evidence, reviews, recovery, conflict preparation, native publication, and legacy Git patch export without treating Git branches or worktrees as the core model.

The first UI purpose is an operator console for review and publication. Success means a technical user can supervise multiple agent work threads, understand review state quickly, recover or revert safely, and publish a result with enough evidence to trust it.

## Brand Personality

Calm, precise, audit-worthy.

Anvics should feel like a serious developer tool built for high-trust work. It should reduce anxiety around agent activity by showing state, provenance, and next actions plainly. It should avoid theatrical AI energy. The product voice is direct, specific, and operational.

## Anti-references

- Do not look like a GitHub clone with agent labels pasted on.
- Do not look like a flashy AI SaaS dashboard.
- Do not use neon terminal aesthetics as a personality substitute.
- Do not use generic beige dashboard styling as the whole visual identity.
- Do not center agents as the UI audience. The UI serves humans supervising agent-native work.
- Do not hide provenance, risk, or recovery behind decorative summaries.

## Design Principles

1. Make agent work legible to humans.
2. Show provenance before persuasion.
3. Prefer operational calm over AI spectacle.
4. Keep Git concepts out of primary navigation.
5. Make recovery and audit paths visible before they are needed.

## Accessibility & Inclusion

Target WCAG AA for the private beta UI. The interface should be keyboard-first, readable at dense information levels, and safe for reduced-motion users. Color must not be the only signal for risk, publication, conflict, or recovery state. Text labels should remain explicit because Anvics users are making trust decisions, not browsing decoration.
